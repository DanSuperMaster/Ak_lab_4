## Язык программирования

Lisp-подобный язык на S-выражениях. Поддерживает рекурсивные функции, ветвления,
циклы, арифметику, строки и потоковый ввод-вывод.

### Синтаксис (БНФ)

```text
<program>        ::= { <top-level-form> }
<top-level-form> ::= <defun> | <expr>

<expr>    ::= <integer> | <string> | <identifier> | 'T' | 'NIL'
            | <setq> | <if> | <while> | <progn> | <cond>
            | <arith> | <compare> | <io> | <list> | <call>

<integer> ::= [ '-' ] <digit> { <digit> }
<string>  ::= '"' { <любой символ кроме '"'> | '""' } '"'
<identifier> ::= <id-char> { <id-char> }   

<setq>    ::= '(' 'SETQ' <identifier> <expr> ')'
<if>      ::= '(' 'IF' <expr> <expr> [ <expr> ] ')'
<while>   ::= '(' 'WHILE' <expr> <expr> ')'
<progn>   ::= '(' 'PROGN' { <expr> } ')'
<cond>    ::= '(' 'COND' { '(' <expr> <expr> ')' } ')'

<arith>   ::= '(' ( '+' | '-' | '*' | '/' | '%' ) <expr> <expr> ')'
<compare> ::= '(' ( '=' | '<' | '>' ) <expr> <expr> ')'

<io>      ::= '(' 'READ' ')'
            | '(' 'WRITE' <expr> ')'
            | '(' 'PEEK'  <expr> ')'

<list>    ::= '(' 'LIST' { <integer> } ')'

<defun>   ::= '(' 'DEFUN' <identifier> '(' { <identifier> } ')' <expr> ')'
<call>    ::= '(' <identifier> { <expr> } ')'
```

### Семантика

| Форма | Значение |
|-------|----------|
| `(SETQ x e)` | вычислить `e`, сохранить в переменную `x` |
| `(IF c a b)` | если `c` ≠ 0 — `a`, иначе `b` (ветка `b` опциональна, по умолчанию `NIL`) |
| `(WHILE c body)` | пока `c` ≠ 0 выполнять `body` |
| `(PROGN e1 … en)` | вычислить по порядку, значение = последнее |
| `(COND (c1 b1) …)` | первая ветвь с истинным условием |
| `(+ - * / %)`, `(= < >)` | бинарные операции; сравнение даёт `1`/`0` |
| `(READ)` | снять один токен из буфера ввода (пустой буфер → останов модели) |
| `(WRITE e)` | вывести младший байт `e` в буфер вывода |
| `(PEEK a)` | прочитать слово памяти данных по адресу `a` |
| `(LIST n…)` | разместить числа `n… 0` в памяти данных, вернуть адрес начала |
| `(DEFUN f (p…) body)` | определить функцию; `(f a…)` — вызов |

- **Стратегия вычислений** — аргументы вычисляются
  слева направо и кладутся на стек данных перед операцией/вызовом.
- **Любое выражение возвращает значение** на вершину стека: арифметика, сравнения,
  `IF`, `COND`, вызов функции. <!-- Это демонстрируется в тесте `factorial`, где `IF`
  используется как выражение — результат рекурсивной ветви и есть возврат функции:
  `(IF (< N 2) 1 (* N (FACT (- N 1))))`. -->
- **Области видимости.** Переменные — статические слоты в памяти данных. При входе
  в функцию её параметры и локальные переменные сохраняются на стеке возвратов
  , а при выходе восстанавливаются.
- **Типизация** — бестиповая: всё представлено 32-битным машинным словом. `T` = 1,
  `NIL` = 0; истинность — «не ноль». Строки и массивы представлены **адресом** их
  начала в памяти данных.
- **Литералы**: целые (в т.ч. отрицательные), строки (cstr), `T`, `NIL`, массивы
  (`LIST`). <!-- Печать строк и чисел реализована **на самом языке** (функции
  `PRINT-STR` / `PRINT-NUM` в тестах) поверх примитивов `WRITE`/`PEEK`.-->

## Организация памяти

```text
        Регистры
+-----------------------------+
| PC   счётчик команд         |
| AR   адресный регистр       |
| IR   = opcode : arg(24 бит) |
| C    флаг АЛУ (нуль/перенос)|
+-----------------------------+

   Стеки (глубина 1024)
+-----------------------------+
| DS  стек данных (операнды)  |
| RS  стек возвратов (PC,     |
|     сохранение пар/локалов) |
+-----------------------------+

   Единая память (фон Нейман), слово 32 бита, адресация по словам
+------------------------------------------------+
| 0           : код (инструкции)                 |  ◄── PC
|   …                                            |
| data_offset : секция данных                    |  ◄── AR
|   c+0 : переменная 1                           |
|   c+1 : переменная 2 …                         |
|   l+0 : строки (cstr) и массивы (LIST)         |
|   …                                            |
+------------------------------------------------+
| 0x7FFFFE : IN_PORT   (memory-mapped input)     |
| 0x7FFFFF : OUT_PORT  (memory-mapped output)    |
+------------------------------------------------+
```

- **Размер слова** — 32 бита. Инструкция: старший байт — опкод, младшие 24 бита —
  знаковый аргумент. Данные — знаковое 32-битное слово.
- **Адресация** — словная (одно слово на адрес). Виды: непосредственная
  (`PUSH arg`), прямая (`LOAD/STORE arg` — в т.ч. порты), косвенная по вершине
  стека (`LOAD_IND`). Управление потоком — абсолютные адреса (`JMP/JZ/CALL`).
- **Гарвард/Нейман.** Логически память едина (фон Нейман): код и данные лежат в
  одном адресном пространстве, данные располагаются сразу за кодом
  (`data_offset = число инструкций`). Выборка инструкции читает то же ОЗУ по `PC`.
- **Ввод-вывод** — memory-mapped: порты `IN_PORT`/`OUT_PORT` перехватываются по
  адресу в модели памяти, доступ к ним — обычными `LOAD`/`STORE`.

Размещение объектов (компиляция / исполнение):

| Объект | Размещение |
|--------|-----------|
| Целочисленный литерал | непосредственно в `PUSH` (24-битный знаковый аргумент). Не влезающие в 24 бита константы (напр. 10⁹) собираются умножением в рантайме — см. `double_precision` |
| Строка (cstr) | в секции данных: по слову на символ + терминатор `0`; одинаковые строки разделяют память (пул). Литерал даёт на стек адрес начала |
| Массив (`LIST`) | в секции данных: числа подряд + терминатор `0`; литерал даёт адрес начала |
| Переменная | один статический слот в секции данных (`alloc_var`), адресуется прямым `LOAD/STORE` |
| Параметр / локальная переменная функции | тот же статический слот, но сохраняется/восстанавливается через RS в прологе/эпилоге (рекурсия) |
| Процедура (`DEFUN`) | код в памяти команд; определение «перепрыгивается» инструкцией `JMP`, вызов — `CALL` (адрес возврата → RS), выход — `RET` |
| Прерывания | отсутствуют — ввод-вывод потоковый (`stream`), без trap |

## Система команд

### Особенности процессора

- **Стековая машина**: операнды берутся с/кладутся на стек данных `DS`; адреса
  возврата и сохранённые переменные — на стеке возвратов `RS`.
- **Типы данных**: 32-битное знаковое целое (оно же символ, адрес, булево).
- **Память**: единая, словная адресация; ввод-вывод memory-mapped.
- **Поток управления**: `JMP` (безусловный), `JZ` (по нулю — флаг `C` от
  `TEST_ZERO`), `CALL`/`RET` через `RS`. Системы прерываний нет.
- **Управление**: микропрограммное (см. раздел «Модель процессора»).

### Кодирование инструкций

Бинарное, одна инструкция — одно 32-битное слово:

```text
 биты 31..24 | биты 23..0
   опкод     | аргумент (знаковый, 24 бита)
```

Бинарный файл: заголовок `n_instr`, `n_data` (по 32 бита), далее инструкции
(`>I`), затем секция данных (`>i`). Текстовый листинг (`--dump`):

```text
<addr> - <HEXCODE> - <mnemonic> <arg>
0036 - 037FFFFF - STORE 8388607
```

### Набор инструкций

`такты = 2 (выборка: AR ← PC; IR ← MEM[AR], PC++) + число микрокоманд опкода`.

| Opcode | Mnemonic | Operand | Тактов | Действие | Описание |
|:------:|:---------|:-------:|:------:|:---------|:---------|
| 0x01 | `PUSH`     | `imm`  | **3** | `DS.push(imm)` | Положить 24-битную константу на стек |
| 0x02 | `LOAD`     | `addr` | **4** | `DS.push(MEM[addr])` | Прямое чтение из памяти (ввод: `addr = IN_PORT`) |
| 0x03 | `STORE`    | `addr` | **4** | `MEM[addr] = DS.pop()` | Запись в память (вывод: `addr = OUT_PORT`) |
| 0x04 | `LOAD_IND` | –      | **4** | `a = DS.pop(); DS.push(MEM[a])` | Косвенное чтение по адресу с вершины стека |
| 0x10 | `ADD`      | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a + b)` | Сложение двух верхних элементов |
| 0x11 | `SUB`      | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a - b)` | Вычитание |
| 0x12 | `MUL`      | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a * b)` | Умножение |
| 0x13 | `DIV`      | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a / b)` | Целочисленное деление |
| 0x14 | `MOD`      | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a % b)` | Остаток от деления |
| 0x20 | `EQ`       | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a == b)` | Равно: на стек `1`/`0` |
| 0x21 | `LT`       | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a < b)` | Меньше: на стек `1`/`0` |
| 0x22 | `GT`       | –      | **3** | `b = DS.pop(); a = DS.pop(); DS.push(a > b)` | Больше: на стек `1`/`0` |
| 0x30 | `JMP`      | `addr` | **3** | `PC = addr` | Безусловный переход |
| 0x31 | `JZ`       | `addr` | **4** | `if DS.pop() == 0: PC = addr` | Переход, если вершина равна нулю |
| 0x32 | `CALL`     | `addr` | **4** | `RS.push(PC); PC = addr` | Вызов функции |
| 0x33 | `RET`      | –      | **3** | `PC = RS.pop()` | Возврат из функции |
| 0x34 | `RS_PUSH`  | –      | **3** | `RS.push(DS.pop())` | Перенос вершины DS на стек возвратов |
| 0x35 | `RS_POP`   | –      | **3** | `DS.push(RS.pop())` | Перенос вершины RS на стек данных |
| 0x50 | `DUP`      | –      | **3** | `DS.push(DS.peek())` | Дублировать вершину стека |
| 0x51 | `POP`      | –      | **3** | `DS.pop()` | Удалить вершину стека |
| 0xFF | `HALT`     | –      | **3** | останов | Остановить тактовый генератор |

Классификация: **Stack**, микропрограммное УУ, бинарное представление,
потактовая точность.

## Транслятор

Консольное приложение `utils/compiler.py`.

**CLI:**

```text
python -m utils.compiler <source.lisp> <output.bin> [--dump <listing.txt>]
```

- вход: исходный код на Lisp;
- выход: бинарный машинный код (`output.bin`), опционально — текстовый листинг.

**Этапы работы:**

```text
исходник ─► Lexer ─► токены ─► Parser ─► AST (списки) ─► Compiler ─► (code, data) ─► bin
```

1. **Лексер** (`utils/lexer.py`): токены — числа (с унарным минусом), строки
   (с экранированием `""`), скобки, идентификаторы.
2. **Парсер**: рекурсивный спуск по S-выражениям → вложенные списки Python.
3. **Компилятор** (`utils/compiler.py`): обход AST, генерация инструкций для
   стековой машины. Адреса переменных/строк/массивов записываются временно как
   отрицательные смещения данных и доращиваются в `finalize()` до абсолютных
   (`n_instr + offset`); прямые ссылки на функции разрешаются там же.
   Литералы (строки, `LIST`) размещаются в секции данных, на стек кладётся адрес.

## Модель процессора

Консольное приложение `machine/processor.py`.

**CLI:**

```text
python -m machine.processor <binary.bin> [--input <строка|файл>] [--log] [--max-log N]
```

- вход: бинарный код и (опц.) файл/строка для имитации ввода;
- выход: данные из процессора + (с `--log`) журнал состояний с финальным дампом.

### DataPath

```text
                 IR.arg ─────────┬───────────────┐
                                 │               │
                             ┌───▼───┐   ┌───┐  (PUSH_ARG)
                  pc_op ────►│  MUX  │   │PC │     │
                             │  PC   │   └─┬─┘     │
                             └───┬───┘     │       │
   PC / IR.arg / DS ─┐          │ +1/arg/RS│       │
                ┌────▼───┐       ▼         │       ▼
                │  MUX   │    ┌──────┐      │  ┌───────────────┐
                │ ar_sel │    │  PC  │◄─────┘  │  Data Stack   │
                └───┬────┘    └──────┘         │   DS  (TOS)   │
                    ▼                          └──┬────────┬───┘
                 ┌─────┐                  operand │        ▲ push
                 │ AR  │                          ▼        │ result
                 └──┬──┘                ┌──────────┴──┐
               addr │                   │     ALU     │── flag C ─►(JZ)
                    ▼                   │ +-*/% = < > │
              ┌───────────┐  data ────► └─────────────┘
              │  MEMORY   │
              │  + MMIO   │◄── store ── (DS top)
              │ IN/OUT buf│
              └───────────┘
   Return Stack (RS): push PC (CALL) / pop → PC (RET) / RS_PUSH / RS_POP
```

**Регистры/узлы:** `PC` (счётчик команд), `AR` (адресный регистр), `IR`
(opcode+arg), `DS` (стек данных), `RS` (стек возвратов), `ALU` с флагом `C`,
единая память с перехватом MMIO-портов.

**Сигналы УУ:** `ar_sel` (источник `AR`: PC/ARG/DS), `ds_op` (push/pop/dup/
push-mem/pop-mem/push-arg/push-rs), `alu_op` (операция и `TEST_ZERO`),
`rs_op` (push PC/DS), `pc_op` (PC ← arg/RS/arg-if-C), `fetch`, `halt`.

### ControlUnit (микропрограммный)

```text
        ┌──────────────────────────────────────────────┐
        │              Control Unit (mc)                │
        │                                               │
   IR.opcode ─►┌─────────────────┐                      │
        │      │ OPCODE→µADDR     │   ┌──────┐           │
        │      │ (таблица         │──►│ MUX  │──► µPC ───┼──► адрес
   flag C ────►│  диспетчеризации)│   │ next │    │     │     ┌─────────────┐
        │      └─────────────────┘   └──▲───┘    │     │     │ микрокодовое│
        │      cond: SEQ/NEXT/           │ +1     │     │     │   ПЗУ (ROM) │
        │      DISPATCH/IF_C/IF_NC ──────┘        └─────┼────►│             │
        │                                               │     └──────┬──────┘
        │            управляющее слово (32 бита) ◄───────┼────────────┘
        └───────────────────┬───────────────────────────┘
                            ▼  ar_sel, ds_op, alu_op, reg_op, rs_op, pc_op, fetch, halt
                          DataPath
```

- Микрокод хранится в **отдельном ПЗУ** `ROM` (генерируется `_build_microcode`),
  каждое слово — набор сигналов, выставляемых **за один такт**.
- Свой счётчик микрокоманд `µPC`; следующий адрес выбирается полем `cond`:
  `SEQ` (µPC+1), `NEXT` (по `next_addr`), `DISPATCH` (по опкоду после выборки),
  `IF_C`/`IF_NC` (по флагу `C`).
- Раскладка управляющего слова и реализация — `common.py`; исполнение —
  `machine/CU.py` (метод `_exec_word` выставляет все сигналы такта).

### Особенности моделирования

- **Потактовое** (`tick`): каждый такт = одно слово микрокода; журнал пишется
  по тактам (регистры, `µPC`, снимок `DS`/`RS`, исполняемая инструкция/микрооп).
- Ввод-вывод **потоковый**: `READ`(=`LOAD IN_PORT`) снимает токен из буфера,
  пустой буфер → `StopSignal` (останов). `WRITE`(=`STORE OUT_PORT`) дописывает
  байт в буфер вывода.
- В конце журнала — финальный дамп: причина останова, `output_buffer` списком
  байт, число тактов/инструкций, дамп памяти данных.

## Тестирование

Golden-тесты лежат в каталоге [`golden/`](golden/): для каждого алгоритма — исходник
`<name>.lisp` и собранный эталон `<name>.yml`, в котором есть исходный код, число
инструкций, **машинный код и данные** (листинг памяти команд и данных), ввод, вывод,
список байт вывода с числом тактов и **журнал работы процессора** (голова ~24 такта,
далее обрезка, хвост с финальным дампом: причина останова, `output_buffer`, такты и
дамп памяти данных).

Эталоны собираются единым прогоном инструментальной цепочки
(транслятор → модель → дамп/журнал) генератором
[`golden/gen_golden.py`](golden/gen_golden.py):

```text
python -m golden.gen_golden --name hello --src golden/hello.lisp
python -m golden.gen_golden --name cat   --src golden/cat.lisp --input-file golden/cat_input.txt
```

Ввод/вывод — потоковый (`stream`): `READ` снимает один токен из входного буфера
(пустой буфер останавливает модель), `WRITE` дописывает байт в буфер вывода. Печать
строк и чисел реализована **в самом исходном коде** через примитивы `WRITE`/`PEEK`
(процессор умеет лишь вывод одного байта), что демонстрируется в тестах ниже.

### Краткое описание разработанных тестов

| Тест | Что проверяет | Ввод → вывод |
|------|---------------|--------------|
| [hello](golden/hello.yml) | печать строкового литерала (cstr) через пользовательскую `PRINT-STR` (цикл `PEEK`+`WRITE`) | — → `Hello, World!` |
| [cat](golden/cat.yml) | потоковый ввод-вывод, останов по концу буфера (вход потенциально бесконечен) | `foo bar\nbaz` → то же |
